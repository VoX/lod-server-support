# MC 1.20.1 Backport — Research Dossier

Status: **complete** — all four lanes reported (§6 harvested from the stopped agent's worktree)
(fabric/common compile survey, paper compile/reobf/Folia survey, ecosystem pins, Java-17 + test-surface audit).

Target: backport the current v0.5.0 feature set (protocol v16, delivery honesty, Paper support,
soak harness) from the `support/mc1.21.11` line to **Minecraft 1.20.1**, on branch `support/mc1.20.1`
(forked from `support/mc1.21.11` @ 00d4194 — it inherits the dual-line release plumbing, split-world
soak handling, stale-world guard, and contract-test widenings).

## 1. Motivation & consumer reality

- Motivation: **GitHub issue #12** (open) — a user asking for 1.20.1 backward compatibility.
- **Voxy has no public MC 1.20.1 build**: zero Modrinth versions (its published game versions:
  1.20.4 [0.1.x-alphas only, pre-`rawIngest`], 1.21.6–1.21.11, 26.1.x); no 1.20.x branch on Voxy's
  GitHub (`master/dev/mc_1215/mc_1217*/mc12110/12111/2622`); no fork on Modrinth covers 1.20.1.
- Consequence: with no `LSSApi` consumer installed, the LSS client does not set
  `CAPABILITY_VOXEL_COLUMNS` and the server ignores it — a stock install is inert.
  **The release notes must state this plainly**: the 1.20.1 line serves issue #12's requester and
  anyone shipping their own `LSSApi` consumer (e.g. a private Voxy port), not stock Voxy users.
- Maintainer decision (2026-07-02): proceed anyway; **Folia support may be dropped on this line
  if needed**.

## 2. Reference implementation: `origin/mc_1.20.1_backport` (external, Abysmalduck)

An external contributor's working 1.20.1 port of the **v0.1.0-era** codebase (fork point `7b6d321`,
MC 1.21.11 / fabric-api 0.140.2 / pre-batch protocol). Not mergeable (ancient base, drops the Paper
module, deletes Tier 3, mod_version left 0.1.0, no tests for the rewrite) — but a proven reference
for the 0.92-era idioms:

- Pins that worked: `minecraft_version=1.20.1`, `fabric_version=0.92.7+1.20.1`,
  `loom_version=1.15-SNAPSHOT` (we will try loom-remap 1.17.3 first — proven on this wrapper),
  loader floor `>=0.14.22`.
- Networking idiom (fabric-api 0.92 — `CustomPacketPayload`/`PayloadTypeRegistry`/`StreamCodec`
  do not exist before 1.20.5): bare `ResourceLocation` channel ids in `LSSNetworking`; payload
  records with static `read(FriendlyByteBuf)` / instance `write(FriendlyByteBuf)`;
  registration through `ServerPlayNetworking`/`ClientPlayNetworking` global receivers.
- It deleted Tier 3 (`LSSClientGameTests`) — the client gametest API does not exist in
  fabric-api 0.92. Expected for our port too.
- **Voxy surface it targets**: identical to ours — `me.cortex.voxy.commonImpl.WorldIdentifier.of(Level)`
  + `VoxelIngestService.rawIngest(worldId, LevelChunkSection, cx, sy, cz, blockLight, skyLight)`
  (Voxy 0.2.13-alpha+ era internals). It declares no Voxy version; the contributor presumably ran a
  self-built Voxy. Keep our bridge unchanged (it fails safe without Voxy).

## 3. Fixed decisions for this line

- **Java 17 bytecode everywhere** (`options.release = 17`, mixin `JAVA_17`): 1.20.1 ships on JRE 17;
  class-file 61 is the ceiling. All Java 18–21 language features/APIs must be rewritten (audit in §8).
- **Protocol v16 unchanged on the wire** — same payload bytes as every other line; only the Fabric
  transport registration changes (raw channels instead of payload types). Paper's
  `PaperPayloadHandler` already speaks raw `FriendlyByteBuf` — the Fabric side converges on the
  same shape.
- **Tier 3 dropped** on this branch (no client gametest API in fabric-api 0.92). CI loses the
  client-gametest job; the Tier-3-only coverage (E2E handshake, ingest-failure recovery,
  consumer dispatch) is accepted as a support-line gap.
- **Golden NBT corpus regenerated on this branch** (registry ids are version-baked; never copy
  fixtures between branches).
- **Tag scheme**: `v0.5.0+mc1.20.1`; Modrinth versions `v0.5.0+fabric+mc1.20.1` /
  `v0.5.0+paper+mc1.20.1`; PREV_TAG glob `v*+mc1.20.1*`; game-versions exactly `1.20.1`.
- **Paper 1.20.1 runtime is spigot-mapped** (mojang-mapped runtime began in 1.20.5) — the release
  shadowJar AND the dev soakShadowJar must be reobfuscated; `release_check.py`'s
  mappings-namespace manifest expectations need adjusting (lane 2 verifying details).
- **plugin.yml `api-version`** must be `'1.20'` (three-part accepted only since 1.20.5) —
  `PluginYmlContractTest`'s lockstep assertion changes to a truncation comparison (the fallback
  designed in the 1.21.11 plan becomes primary here).
- Effort budget (maintainer, 2026-07-02): support lines are **correct, not perfect** — smoke the
  soak scenarios per platform, don't run full gauntlets; checker timing flakes are re-runs.

### fabric/ main: 165 distinct errors in three buckets

**(a) Networking (~80 errors).** `StreamCodec`/`CustomPacketPayload`/`PayloadTypeRegistry` don't exist in 0.92. Affected: all 6 payload classes (same 10-error pattern each), `LSSNetworking` (dissolves — no registration step in 0.92), `PlayerRequestState` + `RequestProcessingService:266,426` + `DirtyColumnBroadcaster:46` (the `CustomPacketPayload` generic seam + `send`), `LSSClientNetworking:29,100`, `LodRequestManager:285`, `LSSClientCommands` (`ClientCommands` → `ClientCommandManager`, same package/statics — only the name).
**MIGRATION CHOICE (verified both ways):** prefer 0.92's typed **`FabricPacket`/`PacketType`** API over raw channels: payloads become `implements FabricPacket { void write(FriendlyByteBuf); PacketType<?> getType(); }` with `PacketType.create(id, Reader::read)`; `ServerPlayNetworking.send(player, packet)` / `registerGlobalReceiver(PacketType<T>, (packet, player, sender) -> …)` mirror today's call shapes; generic seams parameterize on `FabricPacket`. Raw-channel signatures (reference-branch style) also verified as fallback: `registerGlobalReceiver(ResourceLocation, (server, player, listener, buf, sender) -> …)`, `send(player, id, buf)`, `PacketByteBufs.create()`, `PacketSender.sendPacket(id, buf)` (handy for the handshake reply).

**(b) NBT/registry drift (~55 errors) — key fixes (all verified):**
- `NbtSectionSerializer`: `PalettedContainerFactory` absent → `PalettedContainer.codecRW(Block.BLOCK_STATE_REGISTRY, BlockState.CODEC, Strategy.SECTION_STATES, Blocks.AIR.defaultBlockState())` + `codecRO(biomeRegistry.asHolderIdMap(), biomeRegistry.holderByNameCodec(), Strategy.SECTION_BIOMES, defaultBiome)`; default-biome container ctor order is `(IdMap, T, Strategy)`; `ChunkStatus` has no `.status` subpackage (`byName`/`FULL` exist); `lookupOrThrow` → `registryOrThrow`; `getOrThrow(key)` returns `T` → `getHolderOrThrow`; `getList("sections", 10)` plain ListTag; `contains("Y") ? getInt("Y") : MIN_VALUE`; `getByteArray` direct; `getCompound` plain + `contains(name, 10)` gate. `ChunkMap.read(ChunkPos)` unchanged.
- **⚠ SILENT-CORRUPTION TRAP: call `section.recalcBlockCounts()` after codec-parsing** — 1.20.1's codec path doesn't compute counts; without it every disk-read section reports all-air (breaks the all-air sentinel + DirtyContentFilter seeding). The reference branch and vanilla ChunkSerializer both call it. Never a compile error.
- `ChunkGenerationService`: `new TicketType(0,2)` → `TicketType.create("lss_gen", Comparator.comparingLong(ChunkPos::toLong))`; `addTicketWithRadius/removeTicketWithRadius` → `addRegionTicket/removeRegionTicket(type, pos, level, key)` — **the int is a ticket LEVEL, not a radius; value needs re-derivation (vanilla uses 2-3); mirror add/remove args exactly**; `pos.x()/.z()` → fields; `dimension().identifier()` → `.location()`.
- `RequestProcessingService`: `ResourceKey.identifier()` → `location()` (×3); `player.level()` returns plain `Level` → `player.serverLevel()` (×3).
- `SectionSerializer:30` + `ClientColumnProcessor:185`: `getMinSectionY()` → `getMinSection()`. `ClientColumnProcessor` also needs the factory→codec swap (client decode twin).
- `Identifier` → `ResourceLocation` everywhere; **no `parse(String)` on 1.20.1** → `new ResourceLocation(s)` / `tryParse`. `identifier()` → `location()` also in ColumnCacheStore:225, LodRequestManager:532, DirtyColumnBroadcaster:90, ChunkMapSaveHook:33, FabricOffThreadProcessor.
- `LSSServerCommands`: `net.minecraft.server.permissions.Permissions` absent → `source.hasPermission(Commands.LEVEL_GAMEMASTERS)` (int 2).
- `ColumnCacheStore:58`: `Long2LongOpenHashMap.ensureCapacity` is private in fastutil 8.5.9 (1.20.1's) → pre-size via constructor.
- **Sodium config UI cannot port** (sodium 0.5.x is `me.jellysquid.*`; the `net.caffeinemc...config` API is 0.6+): `LSSConfigMenu` + `LSSModMenuIntegration` (19 errors) → drop/exclude on this branch (config stays JSON; ModMenu 7.2.2's `ModMenuApi`/`ConfigScreenFactory` are API-compatible if a plain screen is ever wanted).
- **Mixins are clean**: both accessors + `ChunkMapSaveHook` target `private boolean save(ChunkAccess)` exist unchanged on 1.20.1. `LevelChunkSection` 2-arg ctor + `write(buf)`, `DataLayer`, `SectionPos.of`, `LightLayer`, `FriendlyByteBuf` helpers — all unchanged.

**(c) Java language: zero errors at release=17** on the fabric side besides `Math.clamp` (ClientSessionGate:153-155, LSSClientConfig:23 → `Mth.clamp` or manual).
