# v0.7.1 candidates — deferred findings from the final v0.7.0 six-lens review (2026-07-18)

Findings from the pre-release six-agent review (release machinery / v18 wire / server
pipeline / client state machine / test-gate integrity / docs) that were validated as REAL
but deliberately deferred — each was judged not release-blocking (experimental-path-only,
rare-corner, tooling-only, or nuisance-severity) and riskier to change pre-tag than to ship.
What WAS fixed pre-tag: the client byte-dimension backpressure halt, `DeferredTicketReleases`
per-entry containment, literal wire-identity byte pins (both platforms), the v18 serve-source
decode-value pin, the recursive gametest entrypoint scan, `release_check.py` nested-common /
per-family presence / `discover()` selftest coverage, the A5 comment direction, and the doc
corrections. See the review round in git history (feat/want-set-requests, 2026-07-18).

## Production

1. **Folia hold-release CAS can resurrect a superseded batch** (MEDIUM; Folia experimental —
   no 26.2 Folia exists upstream to even run it). `republishHeldBatch`'s
   `compareAndSet(null, held)` guards "mailbox empty now", not "no offer since the hold": if a
   newer batch B passes THROUGH the mailbox (router takes it) during the one-tick hold of A,
   the republish succeeds and stale A replaces B's backlog. Self-heals in ≤1 s via
   re-declaration EXCEPT when B was the edge-triggered empty backpressure clear (client goes
   silent → resurrected A refills the backlog of a client that declared it cannot ingest).
   Fix shape preserving the pin: an offer-generation counter on `AbstractPlayerRequestState`
   (bump on every `offerIncomingBatch`; pump records it at hold time; republish only if
   unchanged). `PaperRequestProcessingService.holdAndScheduleRegionProbe` +
   `AbstractPlayerRequestState.republishHeldBatch`.
2. **`OffThreadProcessor.shutdown()` interrupt can race the shutdown sentinel** (LOW): the
   post-sentinel `interrupt()` can hit the JLS notify-vs-interrupt window (or, deterministic,
   the 10-error backoff sleep) and skip the sentinel branch that applies queued timestamp
   invalidations before the final cache save → stale `up_to_date` across restart until the
   chunk is re-edited. Fix shape: consume a pending sentinel inside the InterruptedException
   handler, or join once before interrupting. (`OffThreadProcessor` ~:330-356, :938-964.)
3. **Paper/client containment logs unthrottled** (LOW, log-spam vector): Paper's malformed-C2S
   warn/error sites (`PaperPayloadHandler` decode-null warn, `LSSPaperPlugin.dispatchPluginMessage`
   catch) and the client's unknown-batch-response-type warn (`LSSClientNetworking` dispatch)
   log once per hostile frame/entry with no throttle. Route through a rate limiter (the
   processor's `DROP_WARN_INTERVAL_MS` pattern).
4. **Folia shutdown-window WARN storm** (LOW): `PaperChunkGenerationService.completeAsyncLoad`'s
   schedule-failure path lacks the warn-once latch its null-chunk siblings have.
5. **Folia runtime plugin-disable race** (LOW): a region-thread `onChunkReady` can race
   `PaperChunkGenerationService.shutdown()`'s plain-collection clears (CME on a dying plugin).
6. **Client render-distance-shrink annulus** (LOW): rings that fall out of the vanilla view
   circle after a stationary render-distance reduction sit below `confirmedRing` and are not
   re-walked until movement/dirty/reconnect. Detect `viewDistance` shrink → `resetConfirmedRing`.
7. **`ColumnCacheStore.MAX_CACHE_ENTRIES` (2M) vs `MAX_LOD_DISTANCE` (2048)** (LOW corner):
   an effective distance > ~675 chunks can legitimately save more entries than load accepts →
   whole-file discard each session. Align the caps or make save enforce the load bound.
8. **`ClientSessionGate.onJoin` nulls a live manager without teardown** (LOW, precondition
   unverified): if Fabric ever fires JOIN without a preceding DISCONNECT, the old session's
   cache save is skipped. One-line hardening: teardown if non-null on JOIN.
9. **A5 boundary accounting** (diagnosis aid): the two generation-ticket drop paths in
   `drainGenerationTicketRequests` (state gone / dimension mismatch, both platforms) skip
   `miss_dropped` — now documented in CLAUDE.md's flake catalog + `law_A5`'s comment; a
   counter at those sites would close the identity properly.
10. **Dimension-keyed maps never pruned for unloaded datapack dimensions** (LOW):
    `DirtyContentFilter.hashesByDimension` + `DirtyColumnTracker` retain unloaded-dimension
    state for the server's life.

## CI / tooling

11. **release.yml has no Tier-3 guard**: a docs-only release commit gets no build.yml run, and
    release.yml never runs Tier 3 — a tag can ship without Tier 3 having run on the release
    commit (human checklist is the only gate). Candidate: a pre-publish step querying the
    Build workflow's conclusion for `$GITHUB_SHA` (needs `actions: read`).
12. **release.yml Tier 2 has no retry-once** (build.yml's flake containment) — pre-publish
    red, friction only.
13. **release_check glob validation mirrors, doesn't parse, the six workflow glob sites** —
    a future rename can desync a `mc-publish files:` glob and fail mid-publish.
14. **check_soak selftest gaps**: the traffic-floor anti-vacuity guard, the server end-row
    completion gate, and the pass-only arms in several named checks (generation-disabled
    floors, bandwidth-throttle guards, storm generation floor) have no catch fixtures; A5 is
    single-client-sound only (dedup fan-out adds `miss_dropped` with no left-side term if a
    multi-client soak is ever added); `EXPECTED_RUNS` membership unvalidated.
15. **Entrypoint contract residuals**: nested/second top-level classes carrying `@GameTest`
    inside a listed file are invisible; FQ annotation form unmatched.
16. **VSS cosmetics**: Paper VSS `website:` still points at the LSS Modrinth page; the Sodium
    config-screen title is hardcoded "LOD Server Support" in both jars.
